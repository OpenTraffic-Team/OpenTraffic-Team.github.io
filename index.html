Features

核心能力

从仿真到生产全链路支持，内置多算法自适应调度与多层安全机制。

强化学习

多算法自适应

PolicyManager 统一调度 v1 Stable RL、v2 Offline RL、v3 Online RL 三种算法，支持运行时无缝切换策略。

实时决策

毫秒级响应

单步决策延迟 < 100ms，控制频率 1Hz，满足路口信号实时控制需求。

全链路

仿真到生产无缝

支持 CityFlow / SUMO 仿真环境，与 Redis / NATS 生产通信层无缝对接，一套代码多场景运行。

安全机制

多层安全规则

内置 SafeRules 投票机制，当算法输出违反交通规则时自动回退到安全默认策略，确保路口结构化安全。

监控

实时监控告警

内置健康检查与指标上报，支持告警通知，实时推流与性能分析一不错过。

部署

傻瓜化部署

一行命令启动仿真测试，内置示例脱解依赖，支持生产环境 Docker 化部署。

感知 / 多源融合

多源感知融合

视频、雷达、地磁多传感器数据融合，提供高精度实时交通状态感知，覆盖各种天气与光线条件。

通信 / 零配置切换

零配置切换

Redis / NATS / MQTT 多通道支持，仿真与生产环境同一套接口，切换通信后端无需修改算法代码。

算法 / 多算法自适应

多算法自适应调度

PolicyManager 根据环境状态与历史奖励动态选择最优策略，支持 A/B 测试与在线持续学习更新。

安全 / 多层防护

多层安全防护

SafeRules 兜底层确保所有下发指令符合交通法规，异常时自动切换到固定配时保障路口基本通行。

Quick Start

快速开始

三个示例覆盖从最简仿真到完整生产部署的全部场景，git clone 即可运行。

quick_demo.py 最简单的仿真验证示例，5 分钟跑通
simulation_demo.py 完整 CityFlow 仿真，包含多算法切换
production_demo.py 对接 Redis/NATS 的生产环境完整示例
在 GitHub 上查看完整示例 bash 
# 查看所有示例
git clone https://github.com/OpenTraffic-Team/OpenTraffic-Control
cd OpenTraffic-Control/algorithms_sdk/examples

# CityFlow 仿真
python quick_demo.py

# SUMO 仿真
python simulation_demo.py

# 生产环境
python production_demo.py

# 或直接 pip 安装
pip install opentraffic-sdk
python -c "from algorithms_sdk import AlgorithmSDK; print('OK')"
